# 安裝說明 — EnGenius Knowledge Skill（給同事）

讓你的 Claude Code 擁有 EnGenius 產品知識：問產品 / 規格 / 設定 / 法規時，AI 會去查
官方知識庫（Product SpecHub）再回答，並附上來源。

---

## 你需要什麼

1. **Claude Code**（已安裝）
2. **一把 API key**（`sk_live_…`）— 跟管理員（Terrel）索取；他會在後台幫你發一把。
3. macOS 終端機（Terminal）

---

## 三步驟安裝

### ① 安裝 skill（貼這一行到終端機，按 Enter）

```bash
curl -fsSL https://raw.githubusercontent.com/terrelyeh/claude-skills/main/install.sh | bash
```

看到 `✅ engenius-kb` 就代表裝好了。

### ② 設定你的 API key（一次就好）

把下面的 `sk_live_xxx` 換成管理員給你的 key，貼到終端機：

```bash
echo 'export SPECHUB_API_KEY="sk_live_xxx"' >> ~/.zshrc
source ~/.zshrc
```

> 這是寫進你「自己電腦」的設定檔（`~/.zshrc`），不會外流、不會上傳。
> ⚠️ key 不要貼到 Slack / email 群組 / 任何程式碼裡。

驗證有設成功（會印出你的 key）：

```bash
echo $SPECHUB_API_KEY
```

### ③ 重開 Claude Code

關掉再打開 Claude Code（讓它讀到新 skill 和 key）。

---

## 怎麼用

在任何 Claude Code session：

- 直接打指令：
  ```
  /engenius-kb ECW536 支援 WiFi 7 嗎？
  ```
- 或自然發問（提到型號或 EnGenius 產品問題時會自動觸發）：
  ```
  推薦適合飯店的 EnGenius 網路方案
  ECC500 和 ECC500Z 差在哪？
  台灣 6GHz 法規限制？
  ```

AI 會去查知識庫、只根據查到的內容回答，並標出來源。

---

## 常見問題

| 狀況 | 解法 |
|---|---|
| 叫 skill 時說「找不到 SPECHUB_API_KEY」 | 第 ② 步沒做或沒重開終端機。重做 ② 並 `source ~/.zshrc` |
| 回「Invalid API key」 | key 貼錯，跟管理員確認 |
| 回「API key disabled」 | 你的 key 被停用了，找管理員 |
| 回「Rate limit exceeded」 | 短時間問太多，等幾秒再試 |
| **之前能用、突然不能用**（回應變成叫你登入，或收到一堆 HTML 而不是答案） | skill 太舊、還指著搬家前的舊網址。**重跑第 ① 步安裝指令**更新即可（不用動 key），然後重開 Claude Code |
| 想更新 skill | 再跑一次第 ① 步即可 |

---

## 之後要更新 skill

管理員更新後，你只要重跑安裝指令（第 ① 步）就會拉到最新版，不影響你的 key。
